📋 Pré-requisitos

Antes de começar, certifique-se de ter:

  • ✅ Conta no Easypanel.io criada
  • ✅ Conta no GitHub (ou GitLab/Bitbucket)
  • ✅ Repositório Git criado (pode ser privado)
  • ✅ Código do projeto commitado e enviado para o repositório
  • ✅ Acesso administrativo ao Easypanel

🔧 Parte 1: Configurar Repositório Git

1 Preparar o Repositório

Se você ainda não tem um repositório Git ou quer usar o seu próprio:

1.1. Criar Novo Repositório no GitHub

  1. Acesse github.com/new
  2. Preencha o nome do repositório (ex: controle-financeiro)
  3. Escolha se será público ou privado
  4. NÃO marque "Initialize with README" (se já tem código local)
  5. Clique em "Create repository"

1.2. Conectar Repositório Local ao GitHub

No terminal, execute os seguintes comandos:

# Navegar até a pasta do projeto cd controlefinanceiro # Verificar se já é um repositório Git git status # Se não for, inicializar git init # Adicionar todos os arquivos git add . # Fazer o primeiro commit git commit -m "Initial commit - Preparando para deploy" # Adicionar o repositório remoto (substitua SEU-USUARIO e SEU-REPO) git remote add origin https://github.com/SEU-USUARIO/SEU-REPO.git # Enviar código para o GitHub git branch -M main git push -u origin main
Dica: Se o repositório já existe e você quer trocar o remote:
# Ver remote atual git remote -v # Remover remote antigo git remote remove origin # Adicionar novo remote git remote add origin https://github.com/SEU-USUARIO/SEU-REPO.git # Enviar código git push -u origin main

2 Verificar Estrutura do Projeto

Certifique-se de que a estrutura do projeto está correta:

controlefinanceiro/ ├── backend/ │ ├── package.json ← Deve existir! │ ├── server.js │ ├── database/ │ └── ... ├── frontend/ │ ├── package.json ← Deve existir! │ ├── src/ │ └── ... └── README.md

🔑 Parte 2: Gerar Token do GitHub

Importante: Para repositórios privados, você precisa de um Personal Access Token do GitHub.

1 Acessar Configurações do GitHub

  1. Faça login no GitHub
  2. Clique na sua foto de perfil (canto superior direito)
  3. Clique em "Settings"

2 Navegar para Personal Access Tokens

  1. No menu lateral esquerdo, role até o final
  2. Clique em "Developer settings"
  3. No menu lateral, clique em "Personal access tokens"
  4. Clique em "Tokens (classic)"
  5. Clique em "Generate new token" → "Generate new token (classic)"

3 Configurar o Token

Campo Valor
Note Easypanel Deploy ou Easypanel - Controle Financeiro
Expiration Escolha um período (ex: 90 dias) ou "No expiration"
Scopes Marque ✅ repo (Full control of private repositories)

4 Gerar e Copiar Token

  1. Role até o final da página
  2. Clique em "Generate token" (botão verde)
  3. ⚠️ IMPORTANTE: O token será mostrado APENAS UMA VEZ
  4. COPIE O TOKEN IMEDIATAMENTE (começa com ghp_)
  5. Guarde em local seguro (ex: editor de texto temporário)

🗄️ Parte 3: Configurar PostgreSQL no Easypanel

1 Criar Container PostgreSQL

  1. No painel do Easypanel, clique em "New Service" ou "Add Service"
  2. Selecione "PostgreSQL"
  3. Configure:
    • Nome do serviço: postgres ou postgresql
    • Versão: 14 ou 15 (recomendado)
    • Senha: Crie uma senha forte e anote!
    • Database: controle_financeiro ou financialmanagement
  4. Clique em "Create" ou "Deploy"

2 Expor PostgreSQL Externamente

  1. Após criar o PostgreSQL, clique no serviço para abrir as configurações
  2. Vá para a seção "Ports" ou "Networking"
  3. Configure:
    • Porta Interna: 5432 (porta padrão do PostgreSQL)
    • Porta Externa: Escolha uma porta (ex: 5433, 5434)
    • Tipo: TCP
    • Public: ✅ Marque esta opção para permitir acesso externo
  4. Salve as configurações

3 Anotar Informações de Conexão

Você precisará dessas informações para configurar o backend:

Campo Onde encontrar
DB_HOST Hostname do servidor Easypanel (ex: seu-servidor.easypanel.io)
DB_PORT Porta externa que você configurou (ex: 5433)
DB_NAME Nome do banco que você criou (ex: controle_financeiro)
DB_USER Geralmente postgres
DB_PASSWORD A senha que você definiu ao criar o container

🚀 Parte 4: Deploy do Backend

1 Criar Novo Serviço no Easypanel

  1. No painel do Easypanel, clique em "New Service"
  2. Selecione "App" ou "Node.js"
  3. Escolha "GitHub" como fonte

2 Configurar Token do GitHub

Se você ainda não configurou o token:

  1. No Easypanel, vá para "Settings" → "Integrations" → "GitHub"
  2. Cole o token que você copiou (começa com ghp_)
  3. Clique em "Save"

3 Preencher Informações do Repositório

Na tela de criação do serviço, preencha:

Campo Exemplo Descrição
Proprietário Obrigatório joaosilva Nome de usuário ou organização do GitHub (sem @, sem espaços)
Repositório Obrigatório controlefinanceiro Nome do repositório (sem .git, sem espaços)
Ramo Obrigatório main ou master Branch principal do repositório
Caminho de Build Obrigatório /backend Caminho onde está o package.json do backend
Como descobrir essas informações:
  • Se sua URL do GitHub é: https://github.com/joaosilva/controlefinanceiro
  • Então: Proprietário = joaosilva, Repositório = controlefinanceiro
  • Se o package.json está em backend/package.json, use /backend
  • Se está na raiz, use / ou deixe vazio

4 Configurar Build e Start

Na aba "Source" ou "Build":

Campo Valor
Build Method Nixpacks (geralmente detectado automaticamente)
Install Command Deixe vazio (será npm install automaticamente)
Build Command Deixe vazio
Start Command node server.js
Port 5000

5 Configurar Variáveis de Ambiente

Na seção "Environment Variables" ou "Env", adicione:

NODE_ENV=production PORT=5000 # Configurações do PostgreSQL DB_HOST=seu-servidor.easypanel.io DB_PORT=5433 DB_NAME=controle_financeiro DB_USER=postgres DB_PASSWORD=sua-senha-postgres # JWT Secret (gere uma string aleatória segura) JWT_SECRET=sua-chave-secreta-jwt-aqui
⚠️ IMPORTANTE:
  • Use o host externo do PostgreSQL (não localhost)
  • Use a porta externa que você configurou
  • Para gerar um JWT_SECRET seguro, use: openssl rand -base64 32
  • Nunca compartilhe essas variáveis publicamente!

6 Configurar Domínio

  1. Na seção "Domains" ou "Domain":
  2. Adicione seu domínio (ex: api.seudominio.com)
  3. Ou use o domínio fornecido pelo Easypanel (ex: backend.seudominio.easypanel.host)
  4. Marque "Enable SSL" ou "Let's Encrypt" para HTTPS automático

7 Fazer Deploy

  1. Clique em "Save" ou "Deploy"
  2. O Easypanel irá:
    • Clonar o repositório
    • Instalar dependências (npm install)
    • Construir a aplicação
    • Iniciar o servidor
  3. Aguarde o deploy completar (você verá os logs em tempo real)
  4. Verifique os logs para confirmar que o servidor iniciou corretamente

8 Verificar Health Check

Após o deploy, teste o endpoint de health:

# Substitua pela URL do seu backend curl https://seu-backend.easypanel.host/health # Deve retornar: # {"status":"ok","database":"connected","timestamp":"..."}

🎨 Parte 5: Deploy do Frontend

1 Criar Novo Serviço para Frontend

  1. No painel do Easypanel, clique em "New Service"
  2. Selecione "App" ou "Static Site"
  3. Escolha "GitHub" como fonte

2 Configurar Repositório

Preencha as mesmas informações do backend, mas:

Campo Valor
Proprietário Mesmo do backend
Repositório Mesmo do backend
Ramo Mesmo do backend
Caminho de Build /frontend (onde está o package.json do frontend)

3 Configurar Build

Campo Valor
Build Method Nixpacks
Install Command Deixe vazio
Build Command npm run build
Start Command Deixe vazio (será servido como site estático)
Output Directory build (pasta onde o React gera os arquivos estáticos)

4 Configurar Variáveis de Ambiente

Adicione a URL do backend:

REACT_APP_API_URL=https://seu-backend.easypanel.host/api
Importante: Use https:// (não http://) para evitar erros de Mixed Content.

5 Configurar Domínio

  1. Adicione seu domínio (ex: app.seudominio.com)
  2. Ou use o domínio do Easypanel
  3. Habilite SSL/HTTPS

6 Fazer Deploy

  1. Clique em "Save" ou "Deploy"
  2. Aguarde o build completar
  3. O frontend será servido como site estático

📝 Parte 6: Executar Migrações do Banco

1 Acessar Terminal do Backend

  1. No painel do Easypanel, abra o serviço do backend
  2. Vá para a seção "Terminal" ou "Console"
  3. Isso abrirá um terminal no navegador

2 Executar Migrações

No terminal, execute:

# Navegar para o diretório do backend (se necessário) cd backend # Executar migrações npm run migrate # Ou executar diretamente node database/migrate.js
Você deve ver mensagens de sucesso indicando que as tabelas foram criadas.

👤 Parte 7: Criar Usuário Admin

1 Gerar Hash da Senha

No terminal do backend, execute:

cd backend node database/generate_admin_hash.js

Isso gerará um SQL com o hash da senha. Copie o SQL gerado.

2 Executar SQL no PostgreSQL

Você pode executar o SQL de duas formas:

Opção A: Via Terminal do PostgreSQL

  1. No Easypanel, abra o serviço PostgreSQL
  2. Vá para "Terminal"
  3. Execute: psql -U postgres -d controle_financeiro
  4. Cole o SQL gerado

Opção B: Via pgAdmin ou DBeaver

  1. Conecte ao PostgreSQL usando as credenciais
  2. Execute o SQL gerado

3 Testar Login

Acesse o frontend e faça login com:

  • Usuário: admin
  • Senha: A senha que você usou ao gerar o hash (padrão: admin123)

🆘 Troubleshooting

Erro: "Cannot connect to database"

Solução:

  • Verifique se o PostgreSQL está exposto externamente
  • Confirme as credenciais nas variáveis de ambiente
  • Verifique se a porta está correta
  • Teste a conexão externa com um cliente PostgreSQL

Erro: "Repository not found" ou "Access denied"

Solução:

  • Verifique se o token do GitHub está configurado corretamente
  • Confirme que o token tem permissão repo
  • Verifique se o Proprietário e Repositório estão corretos (case-sensitive)
  • Se for organização, o administrador precisa autorizar

Erro: "Build failed" ou "Cannot find package.json"

Solução:

  • Verifique o Caminho de Build
  • Confirme que o package.json está no caminho especificado
  • Se o backend está em backend/, use /backend
  • Se está na raiz, use / ou deixe vazio

Erro: "502 Bad Gateway"

Solução:

  • Verifique se o backend está rodando (veja os logs)
  • Confirme que a porta está correta (5000)
  • Verifique a configuração do domínio (deve apontar para a porta 5000)
  • Verifique se o comando de start está correto: node server.js

Erro: "Mixed Content" no frontend

Solução:

  • Certifique-se de que REACT_APP_API_URL usa https://
  • Não use http:// quando o frontend está em HTTPS
  • Verifique se o backend também está em HTTPS

Backend inicia mas depois para (SIGTERM)

Solução:

  • Verifique os logs do Easypanel
  • Confirme que o comando de start está correto
  • Verifique se todas as variáveis de ambiente estão configuradas
  • Certifique-se de que o backend está escutando em 0.0.0.0 (não localhost)

✅ Checklist Final

Use este checklist para garantir que tudo está configurado corretamente:

  • Código commitado e enviado para o GitHub
  • Token do GitHub gerado e configurado no Easypanel
  • PostgreSQL criado e exposto externamente
  • Informações de conexão do PostgreSQL anotadas
  • Backend criado como novo serviço no Easypanel
  • Repositório configurado corretamente (Proprietário, Repositório, Ramo, Caminho de Build)
  • Variáveis de ambiente do backend configuradas
  • Domínio do backend configurado com SSL
  • Backend deployado e rodando
  • Health check do backend funcionando (/health)
  • Frontend criado como novo serviço no Easypanel
  • Variável REACT_APP_API_URL configurada no frontend
  • Domínio do frontend configurado com SSL
  • Frontend deployado e acessível
  • Migrações do banco executadas
  • Usuário admin criado
  • Login funcionando no frontend
  • Backups do PostgreSQL configurados (recomendado)
